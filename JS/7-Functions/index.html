Arguments
Parameters ( or arguments in JS ) are handled in a number of range. 1 where 2 are required will be NaN

3+ where 2 are required will be handled, but only the first 2
Arguments around parameters are iterable and can be displayed in a loop

Try and Catch
Try and Catch are used for catching errors in code .
This is known as defensive programming ( AKA validation )
try blocks contain code which may cause an exception. If it does, the catch block corrects it
Let Vs Var
let and var are both used, but you should avoid var
let only works in its block of code ( block scoped)
Var works everywhere in a function ( function scopted)
This was changed in ES6 ( aka 2015 )
This in JS
If this is in a method in an object ( function ), then it refers to the function
If this is in an object, then it refers to the global object
Basically, this refers to one up the chain

In a constructor class, it refers to the local version, not the one in the parameters, or the global one
